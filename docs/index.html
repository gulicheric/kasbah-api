üîê Authentication

All API requests (except health checks) require HMAC-SHA256 authentication to ensure security and prevent tampering.

Required Headers

Kasbah-Key: pk_live_your_api_key_here Kasbah-Signature: t=1234567890,s=abc123def456...

Signature Generation

Generate the signature using this payload format:

timestamp + "\n" + method + "\n" + path + "\n" + body

Example (Node.js)

const crypto = require('crypto'); function generateSignature(method, path, body, secret) { const timestamp = Math.floor(Date.now() / 1000); const payload = `${timestamp}\n${method}\n${path}\n${body}`; const signature = crypto .createHmac('sha256', secret) .update(payload, 'utf8') .digest('hex'); return `t=${timestamp},s=${signature}`; }
‚ö†Ô∏è Important: Signatures are only valid for 5 minutes to prevent replay attacks.

GET /v1/orders List Orders

Description: Retrieve orders with filtering and pagination

Authentication: Required

Query Parameters

  • status - Filter by order status (PENDING, APPROVED, FULFILLED, CANCELLED)
  • customer_id - Filter by customer ID
  • supplier_id - Filter by supplier ID
  • created_at[gte] - Orders created after date (ISO-8601)
  • created_at[lte] - Orders created before date (ISO-8601)
  • limit - Results per page (1-200, default: 50)
  • cursor - Pagination cursor
GET /v1/orders?status=FULFILLED&limit=25
GET /v1/orders/{order_id} Get Order Details

Description: Get complete details for a specific order

Authentication: Required

GET /v1/orders/ord_9aK3fQ
GET /v1/orders/{order_id}/items Get Order Items

Description: Get line items for a specific order

Authentication: Required

GET /v1/orders/ord_9aK3fQ/items
GET /v1/orders/{order_id}/events Get Order History

Description: Get status history and events for an order

Authentication: Required

GET /v1/orders/ord_9aK3fQ/events
